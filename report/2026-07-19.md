☆ 趋势一：Agent 记忆架构从“RAG + 向量库”向“LLM 持续整合”范式转移，Google Cloud 的 Always-On Memory Agent 是标志性信号

RAG 的“检索-生成”两阶段架构在 Agent 长时间运行时暴露出上下文碎片化和事实漂移问题。Google 的新参考实现完全抛弃向量数据库，让 LLM 持续读入、关联和整理结构化记忆，这本质上是将“记忆”当作一个实时进程而非静态索引。结合 [teammate-skill](https://www.bestblogs.dev/status/2078328929593737656) 用历史数据模拟人格，以及 Nvidia [Nemotron 3 Embed](https://huggingface.co/blog/nvidia/nemotron-3-embed-wins-rteb) 提升检索质量，Agent 记忆正在从“搜索”走向“主动记忆”。预判：2027 年，Agent 框架将提供两种记忆模式插槽：“RAG 模式”用于快速检索外部知识，“Always-On 模式”用于保持任务上下文，开发者按场景选用。

**为什么今天推荐这篇：** 今天的热点新闻中，NVIDIA 和 Google 在加速 Agent 技术落地，开源模型在激烈竞争，AI 治理在向伦理层延伸。但在这一片“技术加速”的喧嚣中，这篇博客用大量真实（且匿名）的企业案例揭示了 AI 狂热的另一面：高管从未用过 AI 却制定 AI 战略、开发者为了保住饭碗用 AI 重写代码库——这些现象直指 Agent 部署中的“信任幻觉”和“激励扭曲”。与[我们昨天推荐的 Hermit Tech 博客](https://hermit-tech.com/blog/ai-mania-is-eviscerating-global-decisionmaking)形成互补：Hermit Tech 侧重于哲学批判，Nik Suresh 则给出一手企业观察，更具实践参考价值。对于需要判断“我们的 Agent 部署是否健康”的决策者，这篇博客提供的“反面镜子”比技术指南更重要。

- **核心观点一：** 许多企业高层的 AI 战略基于“从没见过 AI 实际应用”的幻想。一位高管在从未用过 ChatGPT 的情况下，为一家 20 亿美元收入的公司制定了完全以 AI 为中心的技术战略。这种决策模式正在系统性降低决策质量。
- **核心观点二：** 内部 Token 排行榜和 AI 使用指标正在扭曲开发者行为。工程师为了在仪表板上保持排名，主动用 AI 重写同事的代码或做不必要的重构，导致净生产力下降——AI 工具从“助手”变成了“绩效工具体系”的一部分。
- **核心观点三：** “用 AI 重写一切”的心态导致系统和数据的熵增。开发团队盲目迁移到 AI 能处理的语言（如 Zig），却忽略了维护成本和稳定性。作者呼吁“审慎的 AI 整合”而非“全盘 AI 化”。
