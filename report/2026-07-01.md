关联：与 Google 在表格数据领域持续的投入（如之前的表格基础模型研究）构成生态闭环，与 Anthropic 的科研 Agent 形成互补——TabFM 解决“数据理解”，Claude Science 解决“工具编排”。

★ 趋势分析

☆ 趋势一：Agent 交互范式正从“聊天对话”进入“任务委托 + 结果审查”时代，聊天界面可能在未来两年内边缘化

Ethan Mollick 的“聊天机器人黄昏论”加上 Claude Science（科研工作台）、Claude Code（编码工作台）、Sonnet 5 的 Agent 能力增强，共同勾勒出一个清晰的趋势：用户不再需要与 AI 聊上十几个来回，而是给一段高层次的指令，然后等待 Agent 交付成果。这对于行业理解“什么是好的 Agent”有根本性影响——好的 Agent 不是“聊得来”的，而是能“交付得了”的。对应的商业模式也将从“按 token 付费”转向“按任务结果付费”。预判：2027 年，所有面向专业用户的 Agent 产品将默认隐藏聊天界面，以仪表盘和交付物审查为主线，聊天功能退回“调试模式”。

关联：前几日报“Claude Tag 原生嵌入工作流”、“Cursor 移动端远程监督”等事件，均指向“少聊天，多监督”的交互模式。

☆ 趋势二：MCP 协议正在从“开发者工具”升级为“平台接口标准”，2027 年可能成为 Agent 时代的 REST API

X 发布官方 MCP 服务器、KDnuggets 的“Qwen3.6 + MCP”教程、以及前几日报“RepoPrompt 采用 MCP 架构”——MCP（Model Context Protocol）正在从实验性协议走向生态标准。与 REST API 不同，MCP 是“模型原生”的——它允许 Agent 主动发现工具能力、动态选择工具、并处理工具调用错误。当 X、OpenAI、Anthropic 等主要玩家都开始支持 MCP 时，Agent 开发将从“为每个工具写定制集成代码”变为“只要实现了 MCP 接口就能被任何 Agent 调用”。预判：2027 年底前，AWS、GCP、Azure 都将原生支持 MCP，MCP 将成为云服务的新一层抽象，决定 Agent 生态的开放度。

关联：与 OpenClaw 的硬件接口标准化、Acti 的键盘 Agent 形成“Agent 生态标准化”的完整图景——硬件层（OpenClaw）、协议层（MCP）、应用层（Acti 键盘）都在走向统一，Agent 开发的门槛系统性降低。

☆ 趋势三：Agent 商业化将驱动“部署服务化”——卖模型/API 不再是终点，卖“部署和优化服务”才是长期利润来源

Amazon 的 10 亿美元 FDE 组织、Anthropic 的 Claude Science 工作台、以及前几日报“火山引擎向生产力场景推手转型”共同说明：模型提供商们意识到，仅仅把模型卖给企业是不够的——企业客户需要“手把手”的 Agent 部署指导。这实际上是在复制云计算行业的演进路径：先是卖基础设施（IaaS/PaaS），发现客户不会用，于是推出专业服务团队（如 AWS Professional Services）。AI 行业正处在同样的转折点——卖 Agent API 是“IaaS”，然后需要“Agent 部署服务”作为 SaaS 层。预判：2026 年下半年至 2027 年，各主要模型提供商都将重组组织架构，设立“Agent 部署服务”部门，从“工程外包”向“Agent 咨询”转型，这将催生一个全新的 Agent 咨询行业。

关联：与之前“企业撤回 AI 投资”的消极信号形成对冲——撤回不是因为 Agent 没价值，而是因为部署门槛太高；Amazon 和 Anthropic 正在建立“降低门槛”的服务体系。

**阅读重点：** 1）通用模型的边际收益递减：当基础能力达到一定程度后，进一步的能力提升需要指数级的计算和数据投入，而垂直领域的专业化可以用更小的模型达到更好的效果；2）专业化的商业模式优势：垂直模型可以收取更高的溢价（客户愿意为“准确率提升 5%”在特定领域支付 10 倍价格），而通用模型面临价格战；3）对 Agent 生态的影响：专业模型将成为 Agent 的“技能模块”，Agent 通过路由/编排在多个专业模型之间切换，而非依赖单一巨型模型。这与 NVIDIA BioNeMo 的策略高度一致。

☆ [The twilight of the chatbots](https://www.oneusefulthing.org/p/the-twilight-of-the-chatbots)
作者：Ethan Mollick · One Useful Thing · 2026-06-30

**为什么今天推荐这篇：** 今天的新闻中，Claude Science 的“工作台”模式、Sonnet 5 的“Agent 能力优先”定价、Acti 的“键盘 Agent”无界面交互——所有事件都在指向同一个核心结论：聊天界面正在成为 AI 交互的历史形态。Ethan Mollick 是 Wharton 商学院教授，过去两年持续跟踪 AI 对教育和工作的影响，他的观点兼具学术严谨性和实践敏锐度。这篇博客直接回答了行业当前最焦虑的问题：“当每个人都学会用了 ChatGPT，然后呢？”答案是：“不再用 ChatGPT 那样的聊天方式。”对于正在构建 Agent 产品的团队，这是理解交互范式转变的必读文章。

- 核心观点一：**聊天界面的成功也是它的局限。** ChatGPT 把 AI 带给大众靠的是“对话”，但对话是“低效的委托”——用户需要用自然语言逐步描述任务、反复修正、等聊天机器人理解。当 Agent 可以自主规划并完成任务时，委托应该是一句话的指令，而不是十轮对话。
- 核心观点二：**自主 Agent 的普及将分三步走：第一步，Agent 成为“副驾驶”（如 Claude Code）；第二步，Agent 成为“自主执行者”（用户只需输入目标）；第三步，Agent 成为“结果交付者”（用户只看最终交付物，不参与过程）。** 目前行业大多处于从第一步到第二步的过渡期，Claude Science 和 Claude Code 已进入第二步。
- 核心观点三：**“去聊天化”将彻底改变产品设计、商业模式和用户教育。** 如果用户不再“聊天”，产品 UI 需要从“对话气泡”改为“任务面板”和“结果审查”；商业模式从“按 token 收费”转为“按完成的任务收费”；用户需要被教育“如何进行好的委托”而非“如何问好的问题”。

适合：AI 产品经理、Agent 创业者、企业技术决策者 · 阅读时间：约 12 分钟
